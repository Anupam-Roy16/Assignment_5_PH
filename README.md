1.What is the difference between var, let, and const?

answer: var, let, and const are used to declare variables in JavaScript, with the key differences being scope, hoisting, and reassignability. var is function-scoped and allows redeclaration, while let and const are block-scoped. let allows reassignment but not redeclaration; const is immutable, allowing neither
2.What is the spread operator (...)?

answer:The spread operator (...) is a syntax in JavaScript and TypeScript that allows an iterable—like an array, string, or object—to be "expanded" into individual elements
3.What is the difference between map(), filter(), and forEach()?

answer:  The primary difference between map(), filter(), and forEach() lies in their return values and their intended use cases. 
map() transforms every element and returns a new array of the same length.
filter() selects specific elements based on a condition and returns a new array (potentially smaller).
forEach() simply executes a function for each element and returns undefined

4.What is an arrow function?

answer: An arrow function is a concise ES6 JavaScript syntax for writing function expressions, characterized by the => operator. They offer a shorter alternative to traditional function expressions, commonly used for callbacks or simple functions because they implicitly return results and have lexical this scoping, capturing context from their surrounding code

5.What are template literals?

answer: Template literals, introduced in ES6, are JavaScript strings defined with backticks (`) that enable easier string interpolation, multiline formatting, and expression embedding. They replace cumbersome string concatenation with ${expression} syntax, improving readability and dynamism.
